# Лабораторная работа 2: Настройка JTA, ПУД (Роли/Привилегии) и Spring Security

## 1. Управление транзакциями (Spring JTA + Jakarta EE JTA)
В приложении настроено использование `JtaTransactionManager` через конфигурационный файл \`config/JtaConfig.java\`. 
Данный менеджер является оберткой над Jakarta EE JTA (используется JTA-менеджер, предоставляемый сервером приложений WildFly). 
Применяется декларативное управление транзакциями с помощью встроенных аннотаций `@Transactional` в сервисном слое (например, в \`EnrollmentService\`).

## 2. Модель разграничения доступа (Роли и Привилегии)
Реализована Role-Based Access Control (RBAC) модель с выделением конкретных привилегий бизнес-логике:

### Список привилегий:
- **`READ_COURSES`**: Чтение списка программ и их тарифов.
- **`WRITE_COURSES`**: Создание новых программ/тарифов или изменение существующих.
- **`APPLY_COURSE`**: Подача заявки на курс (создание enrollment/payment).
- **`MANAGE_APPLICATIONS`**: Просмотр всех поданных заявок в системе.

### Список ролей и их маппинг на привилегии:
1. **`ROLE_USER`** (Обычный студент):
   - `READ_COURSES`
   - `APPLY_COURSE`
2. **`ROLE_TUTOR`** (Преподаватель/Наблюдатель):
   - `READ_COURSES`
   - `MANAGE_APPLICATIONS`
3. **`ROLE_ADMIN`** (Администратор системы):
   - `READ_COURSES`
   - `WRITE_COURSES`

## 3. Spring Security и Аутентификация
В проекте добавлена зависимость `spring-boot-starter-security`.
- **Хранилище УЗ**: Файл `src/main/resources/users.xml`.
- **Сервис аутентификации**: Написан кастомный `XmlUserDetailsService`, который парсит `users.xml`.
- **HTTP Basic**: Включен через конфигурацию `SecurityConfig.java`. Эндпоинты защищены через `authorities()` в цепочке фильтров, проверяя наличие соответствующих привилегий из спецификации выше.
